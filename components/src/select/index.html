Select

The following demos utilize the current control implementation of the Open UI select specification. The implementation is a work in progress but is being used to investigate behavior changes as well as ensure that what is being defined enables the most common scenarios

Note: There are many aspects not currently implemented, but we'll continue to iterate on this.

  • Base component
  • Base with base styles
  • Firefox desktop (Windows)
  • Chromium desktop (Windows)
  • Safari desktop (MacOSX)
  • Bootstrap split button
  • Base with search

Polyfills

Since some aspects of this control are showing capabilities that will need adjustments to the web platform - some items have been polyfilled. Here is a current list that as they become supported we can remove them.

  • oui-option: This is a custom option which enables the capability to allow arbitrary HTML content.
  • checked: Due to the need of a custom option the :checked psuedo class doesn't work so the CSS leverages the checked attribute
